🧠 Primero lo primero — ¿qué estamos haciendo?

Antes de tocar un solo botón, necesitás entender el panorama. Si no entendés POR QUÉ hacés cada paso, cuando algo falle no vas a saber dónde buscar.

¿Qué significa "desplegar" una app?

Ahora tu app corre en tu computadora con python manage.py runserver. Solo vos la ves. "Desplegar" significa ponerla en una computadora conectada a internet 24/7 para que cualquiera con el link pueda usarla.

Pensalo como un restaurante: ahora estás cocinando en tu casa (desarrollo local). Desplegar es abrir un local: necesitás un espacio (servidor), una dirección (dominio), y servicios (base de datos, email, etc).

Las piezas del rompecabezas

Docker — Una caja de mudanza para tu app. Mete TODO lo que necesita (Python, Django, Tailwind) en una "imagen" que puede correr en cualquier computadora. Así evitás el "en mi máquina funciona".
Lightsail Containers — El servicio de AWS que corre tu imagen Docker. Le das la caja, él la pone en un servidor, le asigna una URL, la mantiene viva 24/7, y si se cae la reinicia. Precio fijo: $7/mes. No hay sorpresas en la factura.
Neon (PostgreSQL) — Tu base de datos: donde se guardan usuarios, eventos, ítems. Es PostgreSQL "serverless" — se apaga cuando no se usa y se prende con actividad. Gratis hasta 0.5 GB.
Resend — Tu servicio de correo. Envía emails de verificación y recuperación de contraseña. 3,000 emails/mes gratis.
GitHub Actions (CI/CD) — Tu empleado automático. Cada vez que pusheás a main: corre tests, buildea la imagen, la sube a Lightsail, y despliega. Automático, no tocás nada.
EventBridge + Lambda — Una alarma que cada 5 minutos despierta una función que le dice a tu app "cerrá los eventos vencidos". Costo: $0.
Dominio + DNS — El dominio es la dirección de tu local (hatsik.app). DNS es la guía telefónica que traduce ese nombre a la IP real del servidor.

Paso 1: Configurar AWS CLI

AWS CLI — Un programa para hablar con AWS desde la terminal en vez de clickear botones en la web. Más rápido y reproducible.

1.1 — Instalar

brew install awscli
aws --version

1.2 — Crear usuario IAM (NO uses tu cuenta root)

Tu cuenta root es la llave maestra del edificio. No se la darías al repartidor. Un usuario IAM es una llave que solo abre lo que necesita.
1Abrí IAM: console.aws.amazon.com/iam
2Users → Create user
  • Name: hatsik-deploy
  • NO marques "Console access" — solo es para automatización
3Permisos → "Attach policies directly"

Buscá y tildá: AmazonLightsailFullAccess

¿Por qué solo esta policy? — Con Lightsail Containers no necesitás ECR ni App Runner. Todo (registro de imágenes, servidor, HTTPS) está dentro de Lightsail. Un solo permiso cubre todo.
4Security credentials → Create access key
  • Use case: Command Line Interface (CLI)
  • Descargá el CSV — no se puede volver a ver la Secret Key

1.3 — Configurar CLI

aws configure

# Te pide:
AWS Access Key ID: AKIA............  ← del CSV
AWS Secret Access Key: wJal........  ← del CSV
Default region name: us-east-1
Default output format: json

1.4 — Verificar

aws sts get-caller-identity
# Deberías ver tu Account ID (12 dígitos) — anotalo

Paso 2: Crear la base de datos (Neon)

Base de datos — Donde se guardan los usuarios, eventos, ítems, asignaciones. PostgreSQL es el motor más confiable. Neon te lo da listo, serverless, gratis hasta 0.5 GB.
1Creá cuenta en neon.tech (podés usar GitHub)
2New Project
  • Name: hatsik-production
  • Region: US East (Ohio)
  • PostgreSQL: 16
3Copiá el connection string
postgres://neondb_owner:AbC123@ep-cool-name.us-east-2.aws.neon.tech/neondb?sslmode=require
4Corré las migraciones (creá las tablas)
# Desde la carpeta del proyecto Hatsik:
export DATABASE_URL="postgres://...tu-url-de-neon..."
export SECRET_KEY="temporal-para-migraciones"
export ALLOWED_HOSTS="*"

python manage.py migrate --settings=config.settings.production
python manage.py createsuperuser --settings=config.settings.production
Migraciones — Instrucciones que crean las tablas en la DB. Django las genera automáticamente desde tus modelos. migrate las ejecuta.

Paso 3: Configurar email (Resend)

¿Por qué? — Tu app envía emails de verificación y recovery. No podés enviar directo desde tu servidor porque Gmail/Outlook lo marcarían como spam. Resend se encarga de que lleguen.
1Creá cuenta en resend.com
2Domains → Add Domain → escribí tu dominio

Resend te da registros DNS. Agregalos en tu proveedor de dominio (Namecheap, GoDaddy, Cloudflare).

Registros DNS para email — Le dicen al mundo "Resend tiene permiso de enviar emails en nombre de mi dominio". Es para probar que sos el dueño y evitar suplantación.
3API Keys → Create API Key
  • Name: hatsik-production
  • Permission: Sending access
  • Guardá la key (empieza con re_)

Paso 4: Crear el servicio Lightsail Containers

¿Qué es Lightsail Containers? — El servicio más simple de AWS para correr Docker. Le das una imagen, él la corre, te da una URL con HTTPS, y la mantiene viva 24/7. Precio fijo de $7/mes (plan Nano: 0.25 vCPU, 512 MB RAM). No hay cargos extra por ALB, ni sorpresas. Incluye 500 GB de transfer.
Si App Runner era "alquilar un local llave en mano" (pero lo descontinuaron), Lightsail Containers es "alquilar el local de al lado" — mismo concepto, más barato, y sigue abierto.
1Abrí Lightsail en la consola de AWS

URL: lightsail.aws.amazon.com

2Containers → Create container service

Configuración del servicio

  • Region: Virginia (us-east-1)
  • Capacity: Nano ($7/month) — 0.25 vCPU, 512 MB RAM
  • Scale: 1 (un nodo es suficiente para MVP)
  • Service name: hatsik
¿Plan Nano alcanza? — Sí. Django con Gunicorn (2 workers) corre cómodo en 512 MB para un MVP con pocos usuarios. Si eventualmente necesitás más, subís a Micro ($10, 1 GB RAM) con un click.
NO configures el deployment todavía. Primero creá el servicio vacío (sin imagen). En el siguiente paso vamos a subir la imagen y deployar.
3Click "Create container service"

El servicio se crea en ~1 minuto. Vas a verlo en estado "Ready" sin ningún deployment activo. Esto es normal — ahora vamos a subir tu app.

Paso 5: Subir tu imagen Docker y desplegar

¿Cómo funciona Lightsail Containers? — A diferencia de App Runner (que usaba ECR), Lightsail tiene su PROPIO registro de imágenes integrado. No necesitás ECR. El flujo es: (1) buildeás la imagen en tu máquina, (2) la subís directo a Lightsail con un comando, (3) le decís "usá esta imagen".

5.1 — Instalar el plugin de Lightsail para Docker

# Este plugin permite que AWS CLI suba imágenes a Lightsail:
curl -s "https://s3.us-west-2.amazonaws.com/lightsailctl/latest/darwin-amd64/lightsailctl" \
    -o /usr/local/bin/lightsailctl
chmod +x /usr/local/bin/lightsailctl
¿Qué es lightsailctl? — Es un plugin que le agrega a AWS CLI la capacidad de subir imágenes Docker directamente a tu servicio de Lightsail. Sin este plugin, el comando aws lightsail push-container-image no funciona.

5.2 — Buildear tu imagen Docker

# Desde la carpeta raíz del proyecto Hatsik:
docker build -t hatsik .
Esto tarda 2-5 minutos la primera vez. Docker descarga Python, instala dependencias, compila Tailwind CSS, y deja todo listo. Las siguientes veces es más rápido (cachea lo que no cambió).

5.3 — Subir la imagen a Lightsail

# Sube la imagen directamente al servicio (no necesitás ECR):
aws lightsail push-container-image \
    --region us-east-1 \
    --service-name hatsik \
    --label latest \
    --image hatsik:latest

Cuando termine, te muestra algo así:

Refer to this image as ":hatsik.hatsik.1" in deployments.
Anotá ese nombre de imagen (ej: :hatsik.hatsik.1). Lo necesitás para el siguiente paso. El número al final incrementa cada vez que subís una nueva versión.

5.4 — Deployar la imagen (ponerla a correr)

Ahora le decís a Lightsail "usá esta imagen, con estas variables de entorno":

# Primero generá tus secretos:
python3 -c "
import secrets
print('SECRET_KEY:', secrets.token_urlsafe(50))
print('INTERNAL_CRON_TOKEN:', secrets.token_urlsafe(32))
"

Ahora creá el deployment (reemplazá los valores con los tuyos):

aws lightsail create-container-service-deployment \
    --region us-east-1 \
    --service-name hatsik \
    --containers '{
      "hatsik": {
        "image": ":hatsik.hatsik.1",
        "ports": {"8000": "HTTP"},
        "environment": {
          "DJANGO_SETTINGS_MODULE": "config.settings.production",
          "SECRET_KEY": "TU_SECRET_KEY_GENERADA",
          "DEBUG": "False",
          "ALLOWED_HOSTS": "*.amazonaws.com,tudominio.com",
          "DATABASE_URL": "postgres://...tu-url-de-neon...",
          "RESEND_API_KEY": "re_tu_api_key",
          "RESEND_FROM_DOMAIN": "tudominio.com",
          "INTERNAL_CRON_TOKEN": "TU_CRON_TOKEN_GENERADO"
        }
      }
    }' \
    --public-endpoint '{
      "containerName": "hatsik",
      "containerPort": 8000,
      "healthCheck": {
        "path": "/login/",
        "intervalSeconds": 15,
        "timeoutSeconds": 5,
        "healthyThreshold": 2,
        "unhealthyThreshold": 3
      }
    }'
¿Qué hace este comando?
• containers — define qué imagen correr y con qué variables de entorno
• ports — el puerto donde tu app escucha (Gunicorn en 8000)
• public-endpoint — le dice a Lightsail "exponé este container a internet"
• healthCheck — cada 15 seg, Lightsail visita /login/ para verificar que la app está viva
5Esperá el deploy (3-5 minutos)

Podés ver el estado con:

aws lightsail get-container-services \
    --service-name hatsik \
    --region us-east-1 \
    --query 'containerServices[0].state'

Cuando diga "RUNNING", tu app está en el aire. 🎉

6Obtené tu URL
aws lightsail get-container-services \
    --service-name hatsik \
    --region us-east-1 \
    --query 'containerServices[0].url' \
    --output text

Te da algo como:

https://hatsik.xxxxxxxx.us-east-1.cs.amazonlightsail.com

Abrila en tu navegador — deberías ver la pantalla de login de Hatsik con HTTPS. 🔒

Paso 6: Conectar tu dominio

Dominio — La URL que te da Lightsail funciona pero es larga y fea. Con un dominio propio (tudominio.com) tus usuarios tienen una dirección profesional. Lightsail maneja el HTTPS automáticamente.
1Habilitá el dominio custom en Lightsail

En la consola de Lightsail → tu servicio → pestaña "Custom domains" → "Enable custom domains".

Lightsail va a crear un certificado SSL automáticamente. Te va a pedir que valides la propiedad del dominio agregando un registro CNAME.
2Agregá tu dominio

Click "Create certificate" → escribí tu dominio (ej: tudominio.com).

Lightsail te da un registro DNS de validación:

# Para validar que sos el dueño:
Tipo:   CNAME
Nombre: _abc123.tudominio.com
Valor:  _xyz789.acm-validations.aws.
3Agregá los registros DNS en tu proveedor de dominio

Necesitás DOS registros: uno de validación (el de arriba) y uno para apuntar el tráfico:

# Para apuntar tu dominio a Lightsail:
Tipo:   CNAME
Nombre: tudominio.com  (o @ según tu proveedor)
Valor:  hatsik.xxxxxxxx.us-east-1.cs.amazonlightsail.com
CNAME — Un alias de DNS. Dice "cuando alguien busque mi dominio, mandalo a esta dirección de Lightsail". Como poner un cartel en una calle que dice "el restaurante está a 2 cuadras →".
Cloudflare: Desactivá el proxy (nube gris, no naranja) para que la validación funcione.
4Esperá la validación (5-30 min normalmente)

Cuando el certificado esté "Valid", habilitá el dominio en tu servicio. Listo — https://tudominio.com ahora muestra Hatsik. 🔒

Paso 8: Crear el cron job (EventBridge + Lambda)

¿Para qué? — Hatsik necesita cerrar automáticamente los eventos cuyo deadline ya pasó. Cada 5 minutos, una alarma despierta una función que le dice a tu app "revisá y cerrá lo que haya vencido".
1Creá la función Lambda

Lambda console → Create function:

  • Name: hatsik-cron-trigger
  • Runtime: Python 3.12
  • Architecture: arm64
2Pegá este código:
import json
import urllib.request
import os

def lambda_handler(event, context):
    url = os.environ["APP_URL"]
    token = os.environ["CRON_TOKEN"]

    req = urllib.request.Request(
        f"{url}/internal/close-expired-events/",
        method="POST",
        headers={"X-Internal-Token": token, "Content-Type": "application/json"},
        data=b"{}",
    )

    try:
        with urllib.request.urlopen(req, timeout=30) as resp:
            body = json.loads(resp.read())
            print(f"OK — cerrados: {body.get('closed_count', 0)}")
            return {"statusCode": 200, "body": body}
    except Exception as e:
        print(f"Error: {e}")
        return {"statusCode": 500, "body": str(e)}
3Variables de entorno de Lambda
  • APP_URL = https://tudominio.com
  • CRON_TOKEN = (mismo valor que INTERNAL_CRON_TOKEN)
4Creá el Schedule en EventBridge

EventBridge Scheduler → Create schedule:

  • Name: hatsik-close-expired-events
  • Type: Recurring → Rate: 5 minutes
  • Target: AWS Lambda → Invoke → hatsik-cron-trigger
Costo: $0. Lambda tiene 1 millón de ejecuciones gratis/mes. Vos usás ~8,640.

Paso 9: Verificar que todo funciona

Si algo no funciona

🔴 App no carga

  • Lightsail → tu servicio → "Logs" → mirá los errores
  • Causas comunes: SECRET_KEY faltante, DATABASE_URL mal escrita

🔴 Emails no llegan

  • Dominio en Resend debe estar "Verified"
  • RESEND_FROM_DOMAIN debe coincidir exactamente

🔴 Dominio no funciona

  • dig tudominio.com CNAME — ¿apunta a Lightsail?
  • Certificado debe estar "Valid" en Lightsail

🔴 Cron no cierra eventos

  • Lambda → Monitor → CloudWatch logs
  • Error 401 = CRON_TOKEN no coincide con INTERNAL_CRON_TOKEN

Paso 10: Seguridad final

Activá MFA en tu cuenta root de AWS AHORA. Es lo más importante. Si alguien te roba la contraseña, sin MFA tiene acceso total a todo.
