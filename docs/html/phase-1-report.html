    What Was Built

    Django Project Structure

    • Split settings: base.py, local.py, production.py
    • python-decouple for env management, dj-database-url for DB parsing
    • Function-based views with django-htmx middleware
    • WhiteNoise for production static file serving

    Database (PostgreSQL)

    • 8 tables from DATABASE_SCHEMA.md: users, tokens (2), events, participations, items, assignments, suggestions
    • 5 enums as TextChoices: event_status, event_role, access_status, suggestion_status, item_unit
    • Custom User model (AbstractBaseUser, UUID pk, email identifier)
    • All indexes and constraints including partial unique on item_assignments

    Tailwind CSS v4

    • Standalone CLI binary (no Node.js dependency)
    • Hatsik design tokens via @theme: colors (#E8432D coral), spacing, radii, fonts
    • Makefile targets: tailwind-watch, tailwind-build, tailwind-install

    Base Templates & UI Components

    • base.html with Google Fonts (Nunito + Plus Jakarta Sans), Tailwind CSS, HTMX
    • 6 components: _button, _card, _badge, _modal, _input, _toast
    • Dev component gallery at /dev/components/

    Email & Infrastructure

    • Resend SDK with send_verification_email and send_password_reset_email
    • Docker + docker-compose (Django + PostgreSQL 16)
    • GitHub Actions CI/CD: ruff lint → test → Docker build → ECR → App Runner
    • Internal cron endpoint with token auth for EventBridge
