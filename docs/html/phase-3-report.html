✅ Criterios de Finalización (ROADMAP)

  • Usuario puede crear un evento y verlo en el Dashboard
  • Dashboard muestra rol y status correctos para cada evento
  • Link compartido y QR descargable se generan correctamente
  • Owner puede cerrar, reabrir y cancelar el evento (con confirmación)
  • Owner puede asignar y revocar co-admins (revocación requiere confirmación)
  • Owner puede remover participantes (bloqueado si tienen asignaciones compradas)
  • Cierre automático se ejecuta cuando se alcanza assignment_deadline_at
  • Los 5 screens de Stitch para esta fase coinciden con el design system

Migración del Design System

Se migró completamente de tokens custom a Material Design 3 alineado con Stitch. Esto afectó:

  • input.css — Paleta M3 completa (primary #b51d0c, secondary #a93624, surface-container-*, state colors)
  • base.html — Sidebar + TopBar + Mobile drawer unificado
  • auth-base.html — Material Symbols Outlined + focus rings
  • 6 templates de auth refactorizados para consistencia visual

⚠️ Nueva regla establecida

Stitch Design Fidelity (MANDATORY) — Todo template/UI con screen en Stitch DEBE construirse desde el HTML source code descargado de Stitch. Nunca inventar layout ni estilos. Ver .kiro/steering/tech.md para el proceso completo.
