Fase 3 — Events & Dashboard

Un usuario verificado puede crear eventos, ver sus eventos, gestionar el ciclo de vida, compartir link/QR, administrar co-admins y remover participantes.

🛠️ Requisitos previos

Preparar el entorno

  1. Asegurarse de tener el virtualenv activo: source .venv/bin/activate
  2. Verificar que PostgreSQL esté corriendo (local o Neon)
  3. Correr migraciones: python manage.py migrate
  4. Iniciar el servidor de desarrollo: python manage.py runserver
  5. Abrir el navegador en http://localhost:8000
  6. Tener un usuario registrado y verificado (completar Fase 2 antes)
  7. Estar logueado con ese usuario verificado
💡 Todos los flujos de esta fase requieren un usuario verificado y logueado, salvo la Ficha Pública que se prueba también sin autenticación.

📋 1. Dashboard (Mis Eventos)

Empty state

Precondiciones: Usuario verificado sin eventos creados ni participaciones activas.

  1. Navegar a http://localhost:8000/events/
  2. Esperado: Se muestra un empty state indicando que no hay eventos
  3. Verificar que existe un CTA para crear un evento nuevo

Listado con eventos

Precondiciones: Usuario con al menos un evento creado.

  1. Navegar a http://localhost:8000/events/
  2. Esperado: Se muestran los eventos del usuario
  3. Verificar que cada evento muestra el rol del usuario (Owner, Co-admin, Participante)
  4. Verificar que cada evento muestra el status correcto (active, closed, cancelled)

Filtrado de participaciones inactivas

Precondiciones: Usuario que fue removido de un evento o dejó un evento.

  1. Navegar a http://localhost:8000/events/
  2. Esperado: Eventos con status de participación pending, removed o left NO aparecen en el listado
  3. Verificar en la base de datos que los registros existen pero no se muestran

➕ 2. Crear Evento

Creación exitosa (happy path)

Precondiciones: Usuario verificado y logueado.

  1. Navegar a http://localhost:8000/events/create/
  2. Verificar que se muestra el formulario con campos: nombre, fecha del evento
  3. Completar: Nombre = "Amigo Invisible 2025", Fecha = fecha futura
  4. Click en "Crear evento"
  5. Esperado: Redirige al detalle del evento recién creado
  6. Navegar a /events/
  7. Esperado: El evento aparece en el dashboard con rol "Owner"

Validación de campos obligatorios

  1. Navegar a http://localhost:8000/events/create/
  2. Enviar el formulario sin nombre
  3. Esperado: Error de validación indicando que el nombre es obligatorio
  4. Enviar el formulario sin fecha
  5. Esperado: Error de validación indicando que la fecha es obligatoria

Campos opcionales

  1. Navegar a http://localhost:8000/events/create/
  2. Completar nombre y fecha, además agregar descripción y fecha límite de asignación
  3. Click en "Crear evento"
  4. Esperado: Se crea correctamente con todos los campos opcionales guardados
  5. Verificar en el detalle que descripción y fecha límite se muestran

🔍 3. Detalle del Evento

Vista del Owner

Precondiciones: Ser el Owner del evento.

  1. Navegar al detalle de un evento propio: /events/<id>/
  2. Esperado: Muestra nombre, fecha, status, lista de participantes
  3. Verificar que se muestran los botones de lifecycle (cerrar, cancelar)
  4. Verificar que se muestran opciones de gestión de participantes

Acceso denegado a no-participante

Precondiciones: Estar logueado con un usuario que NO es participante del evento.

  1. Navegar al detalle de un evento del cual no sos participante
  2. Esperado: Recibe error 403 Forbidden

✏️ 4. Editar Evento

Edición exitosa como Owner

Precondiciones: Ser Owner del evento.

  1. Navegar a /events/<id>/edit/
  2. Cambiar el nombre del evento y la fecha
  3. Click en "Guardar cambios"
  4. Esperado: Se guardan los cambios correctamente
  5. Verificar en el detalle que los cambios se reflejan

Acceso denegado a no-owner

Precondiciones: Estar logueado como participante (no Owner) del evento.

  1. Navegar a /events/<id>/edit/ de un evento donde NO sos Owner
  2. Esperado: Recibe error 403 Forbidden

🔄 5. Ciclo de Vida

Cerrar evento

Precondiciones: Ser Owner de un evento con status active.

  1. Desde el detalle del evento, ejecutar acción de cerrar (POST /events/<id>/close/)
  2. Esperado: El status del evento cambia a "closed"
  3. Verificar en el dashboard que el evento muestra status "closed"

Reabrir evento

Precondiciones: Ser Owner de un evento con status closed.

  1. Desde el detalle del evento cerrado, ejecutar acción de reabrir (POST /events/<id>/reopen/)
  2. Esperado: El status del evento vuelve a "active"
  3. Verificar en el dashboard que el evento muestra status "active"

Cancelar evento

Precondiciones: Ser Owner de un evento con status active.

  1. Desde el detalle del evento, ejecutar acción de cancelar (POST /events/<id>/cancel/)
  2. Esperado: El status del evento cambia a "cancelled"
  3. Verificar en el dashboard que el evento muestra status "cancelled"

Reabrir evento cancelado — debe fallar

Precondiciones: Evento con status cancelled.

  1. Intentar reabrir un evento cancelado (POST /events/<id>/reopen/)
  2. Esperado: Error — no se puede reabrir un evento cancelado
  3. Verificar que el status sigue siendo "cancelled"
⚠️ Un evento cancelado es un estado terminal. Solo se puede reabrir un evento "closed", no uno "cancelled".

👥 6. Gestión de Participantes

Promover a co-admin

Precondiciones: Ser Owner. Tener al menos un participante con rol "participant".

  1. Desde el detalle del evento, buscar un participante con rol "participant"
  2. Ejecutar la acción de promover a co-admin
  3. Esperado: El rol del participante cambia a "co_admin"
  4. Verificar que el cambio se refleja en la lista de participantes

Degradar co-admin a participante

Precondiciones: Ser Owner. Tener al menos un co-admin.

  1. Desde el detalle del evento, buscar un co-admin
  2. Ejecutar la acción de degradar a participante
  3. Esperado: El rol del co-admin vuelve a "participant"

Remover participante

Precondiciones: Ser Owner. Participante sin asignaciones compradas.

  1. Desde el detalle del evento, seleccionar un participante
  2. Ejecutar la acción de remover
  3. Esperado: El participante desaparece de la lista activa
  4. Verificar que el participante removido ya no ve el evento en su dashboard

Intentar removerse a sí mismo — debe fallar

Precondiciones: Ser Owner del evento.

  1. Intentar ejecutar la acción de remover sobre tu propio usuario
  2. Esperado: Error — no podés removerte a vos mismo del evento

🔗 7. Compartir Evento

Página de compartir

Precondiciones: Ser Owner o co-admin del evento.

  1. Navegar a /events/<id>/share/
  2. Esperado: Se muestra el link de invitación compartido
  3. Verificar que el link tiene formato /events/join/<token>/

Copiar link al clipboard

  1. En la página de compartir, click en el botón "Copiar link"
  2. Esperado: El link se copia al portapapeles (Clipboard API)
  3. Verificar pegando en otra pestaña que el link es correcto
💡 La Clipboard API requiere HTTPS o localhost. En desarrollo local debería funcionar.

QR code inline

  1. En la página de compartir, verificar que se muestra un código QR
  2. Esperado: El QR se renderiza inline (no como link externo)
  3. Escanear el QR con un celular
  4. Esperado: Abre la URL de join del evento

Descargar QR como PNG

  1. Click en el botón "Descargar QR"
  2. Esperado: Se descarga un archivo PNG con el código QR
  3. Abrir el PNG descargado y verificar que es legible

🌐 8. Ficha Pública del Evento

Vista como no-participante logueado

Precondiciones: Estar logueado con un usuario que NO es participante del evento.

  1. Navegar a /events/join/<token>/
  2. Esperado: Muestra nombre, descripción, fecha y status del evento
  3. Verificar que NO muestra la lista de ítems/regalos
  4. Verificar que hay un botón para unirse al evento

Redirección si ya es participante

Precondiciones: Estar logueado como participante activo del evento.

  1. Navegar a /events/join/<token>/ del evento donde ya participás
  2. Esperado: Redirige automáticamente al detalle del evento

Redirección a login sin autenticación

Precondiciones: No estar logueado.

  1. Cerrar sesión
  2. Navegar a /events/join/<token>/
  3. Esperado: Redirige a login con ?next=/events/join/<token>/

⏰ 9. Cierre Automático

Cierre por deadline expirado

Precondiciones: Crear un evento con assignment_deadline_at en el pasado.

  1. Crear un evento con fecha límite de asignación en el pasado (o modificar uno existente en la DB)
  2. Verificar que el evento tiene status "active"
  3. Ejecutar: curl -X POST http://localhost:8000/internal/close-expired-events/ -H "X-Internal-Token: <token>"
  4. Esperado: El evento cambia a status "closed"
  5. Verificar en el dashboard que el status se actualizó

Sin token — acceso denegado

  1. Ejecutar: curl -X POST http://localhost:8000/internal/close-expired-events/ (sin header)
  2. Esperado: Recibe error 401 Unauthorized
  3. Ejecutar con token incorrecto: -H "X-Internal-Token: token-falso"
  4. Esperado: Recibe error 401 Unauthorized

⚠️ Casos límite

Evento sin descripción

  1. Crear un evento sin completar el campo de descripción
  2. Navegar al detalle del evento
  3. Esperado: No se rompe nada — la UI maneja el campo vacío sin errores

Evento sin deadline

  1. Crear un evento sin fecha límite de asignación
  2. Ejecutar el endpoint de cierre automático
  3. Esperado: El evento NO se cierra automáticamente (solo se cierran los que tienen deadline expirado)

Múltiples participantes con distintos roles

  1. Crear un evento con 3+ participantes: Owner, Co-admin, Participante
  2. Verificar que cada uno ve las acciones correspondientes a su rol
  3. Verificar que Co-admin puede gestionar participantes pero no cambiar lifecycle
  4. Verificar que Participante solo puede ver, no gestionar

Evento cancelado es estado terminal

  1. Tener un evento con status "cancelled"
  2. Intentar cualquier acción de lifecycle (reabrir, cerrar)
  3. Esperado: Todas las acciones fallan — cancelled es irreversible

✅ Checklist final — Done Criteria del ROADMAP

Estos son los criterios de aceptación definidos en docs/ROADMAP.md para la Fase 3.

  • Usuario puede crear evento y verlo en Dashboard
  • Dashboard muestra rol y status correctos
  • Link compartido y QR descargable funcionan
  • Owner puede cerrar, reabrir y cancelar (con confirmación)
  • Owner puede asignar y revocar co-admins
  • Owner puede remover participantes (bloqueado si tienen asignaciones compradas)
  • Cierre automático funciona cuando se alcanza assignment_deadline_at
  • Los 5 screens de Stitch coinciden con el design system
