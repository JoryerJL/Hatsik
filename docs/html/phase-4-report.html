Event Access

Objetivo: Un usuario puede unirse a un evento vía link/QR, pasar por el flujo de aprobación, y ser aceptado o rechazado. También puede abandonar un evento voluntariamente.
Módulo: EventAccess
Fecha: 12 de Julio, 2026
Branch: phase-4/event-access

🏗️ Lo que se construyó

Flujo de acceso a eventos

Implementación completa del módulo EventAccess: solicitud de ingreso, gestión de solicitudes (aceptar/rechazar/corregir), abandono voluntario, y badge de solicitudes pendientes en el dashboard.

  • Join flow: Usuario sigue link → ve Ficha del Evento → solicita unirse → estado pendiente
  • Request management: Owner/Co-admin acepta o rechaza solicitudes (con HTMX)
  • Rejection correction: Owner/Co-admin puede corregir un rechazo erróneo
  • Leave event: Participante abandona voluntariamente (con validación de compras)
  • Dashboard badge: Badge coral con conteo de solicitudes pendientes
  • Redesign event-detail: Hero image, progress bar, grid layout fiel a Stitch

✅ Criterios de finalización (ROADMAP)

  • Usuario siguiendo share link llega a Ficha del Evento
  • Usuario no autenticado es redirigido a login y retorna al evento
  • Se crea la solicitud de ingreso y el usuario ve la pantalla de pendiente
  • La lista de ítems está oculta para usuarios pendientes y rechazados
  • Owner y Co-admin pueden aprobar y rechazar solicitudes
  • Rechazo requiere confirmación. Se puede corregir un rechazo a aceptado
  • Usuario rechazado no puede volver a solicitar (salvo corrección manual)
  • Participante puede abandonar voluntariamente (con confirmación), bloqueado si tiene compras
  • Rol de Co-admin se revoca automáticamente al abandonar
  • Ambas pantallas de Stitch para esta fase coinciden con el design system
